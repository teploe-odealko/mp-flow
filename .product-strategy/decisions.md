# Decision Log

## 2026-03-03 -- Модель монетизации: Open Core + Usage Credits

**Context**: Нужна устойчивая, простая модель монетизации для solo-разработчика. Текущая: подписка 990 руб/мес (на лендинге), без usage billing.

**Options Considered**:
1. Подписка 300 руб/мес (flat) -- слишком дешево, нужно 1000+ клиентов для минимального дохода
2. Тарифные сетки (Starter/Pro/Enterprise) -- слишком сложно для одного разработчика
3. Платные плагины (per-plugin pricing) -- nickel-and-diming, сложный биллинг
4. Чистый usage-based -- непредсказуемо для пользователя, сложная реализация
5. **Open Core + Usage Credits** -- фиксированная подписка + кредиты на дорогие API

**Decision**: Вариант 5 -- Open Core + Usage Credits

**Structure**:
- Self-host: бесплатно, MIT, все функции, свои API ключи
- Cloud: 990 руб/мес, все включено + 500 кредитов/мес
- Кредиты: 1000 шт = 290 руб, 5000 = 990 руб, 20000 = 2900 руб
- Расход: 1688 запрос = 1 кредит, AI фото = 5-10, AI описание = 2

**Reasoning**:
- Простота для пользователя (одна цена, все включено)
- Справедливость (платишь за дорогие операции)
- Простота реализации (2 таблицы, subscriptionMiddleware уже есть)
- Open source совместимость (self-hosted = свои ключи)
- Конкурентное позиционирование (3x дешевле MPStats при большем функционале)
