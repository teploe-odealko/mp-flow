# MPFlow — Полный анализ продукта

## Что такое MPFlow?

**MPFlow** — это открытая AI Agent First система управления маркетплейсами, специализированная на Ozon FBO. Позиционируется как ERP для продавцов на маркетплейсах с встроенной интеграцией через AI-агентов (MCP протокол).

### Ключевой слоган
> "AI-агент управляет магазином за вас — подключите Claude Code, Codex, Cursor, Windsurf или любой MCP-клиент. 54+ инструментов: цены, остатки, аналитика, заказы и весь Ozon API."

## 1. Что делает UI — Функциональность продукта

### Основные страницы и их возможности:

#### 1.1 **Каталог** (`/catalog`)
- Список всех товаров с основными параметрами:
  - Название товара
  - Закупочная цена (с валютой)
  - Текущий остаток на складе
  - **Себестоимость** (средняя цена партии с распределением затрат)
- Клик на товар открывает детальную карточку с:
  - Историей закупок по партиям
  - Распределением затрат (логистика, таможня, упаковка)
  - Связью с Ozon (если товар синхронизирован)
  - Финансовой историей

#### 1.2 **Склад (Warehouse)** (`/warehouse`)
- Состояние складских остатков по каждому товару:
  - Текущие остатки (по источникам: warehouse, in_transit, etc.)
  - Историческая дебет/кредит (поступления, продажи, списания)
  - Движения по каждой ячейке

#### 1.3 **Закупки (Procurement)** (`/procurement`)
**Самая впечатляющая фича — автоматический прогноз закупок:**
- Анализирует историю продаж за период (configurable)
- Рассчитывает дневной rate of sale для каждого товара
- Показывает временную шкалу:
  - Когда закончатся текущие остатки
  - Сколько дней покрытия запасом (с учётом lead time)
- Красиво визуализирует:
  - Красный бар — критический уровень (меньше времени доставки)
  - Жёлтый — требует заказа
  - Зелёный — в норме
- Автоматически рассчитывает нужное количество для заказа
- **"Закупки в 1 клик"** — создание заказа поставщику с одного клика на основе прогноза

#### 1.4 **Поставщики (Suppliers)** (`/suppliers`)
- Список всех заказов поставщикам:
  - Черновик → Заказан → В пути → Принят
- Поддержка распределения общих затрат:
  - По цене товара (пропорционально)
  - По весу (для доставки)
  - Поровну
- Поступление товара в систему с автоматическим расчётом себестоимости каждой единицы

#### 1.5 **Продажи (Sales)** (`/sales`)
- Список продаж с фильтрацией:
  - По статусу (активна, доставлена, возврат)
  - По каналу (Ozon, ручная запись)
  - По дате
- Синхронизация с Ozon FBO (автоматическая):
  - Отправления
  - Возвраты с причинами
  - Комиссии и логистика

#### 1.6 **Финансы (Finance)** (`/finance`)
- **ДДС (Движение Денежных Средств)** — лучше чем в большинстве ERP:
  - Суммарный отчёт по периоду
  - Фильтрация по типам транзакций:
    - Выплата Ozon
    - Оплата поставщику
    - Выручка продаж
    - Доставка
    - Возврат
    - Корректировки
  - Источник: ручная запись, Ozon API, система
  - Ручное редактирование для коррекции
  - Полная история с возможностью удаления

#### 1.7 **Аналитика (Analytics)** (`/analytics`)
**Три типа отчётов:**

**a) P&L (Прибыль и убыток):**
- Выручка → Себестоимость → Комиссии → Операционная прибыль → Чистая прибыль
- Марж по товарам и каналам
- Разбивка комиссий:
  - FBO-услуги
  - Маркетинг
  - Логистика маркетплейса
- Распределение по каналам продаж

**b) Unit Economics (Юнит-экономика):**
- По каждому SKU:
  - Количество продаж
  - Выручка
  - Себестоимость (COGS)
  - Комиссии (по типам)
  - Прибыль и маржа
  - **ROI** — возврат на инвестиции
  - Эффективность каждого товара

**c) Stock Valuation (Стоимость остатков):**
- Текущее значение всех товаров на складе
- Себестоимость каждого товара
- Сортировка для АБС-анализа

#### 1.8 **Плагины (Plugins)** (`/plugins`)
- Управление подключенными плагинами
- Включение/отключение функций
- Конфигурация каждого плагина

#### 1.9 **Настройки (Settings)** (`/settings`)
- Управление учётной записью
- Настройки склада (lead time, coverage days, lookback period)
- API-ключи для интеграций

#### 1.10 **Биллинг (Billing)** (`/billing`)
- Тариф подписки (в облачной версии)
- Лимиты использования
- История платежей

## 3. Что делает Ozon-плагин конкретно

### 3.1 Синхронизация данных

**Входящие потоки (Ozon → MPFlow):**
1. **Товары** (`sync-ozon-products`)
   - Синхронизирует карточки с Ozon
   - Привязывает к MPFlow Master Cards
   - Обновляет цены, описания, характеристики

2. **Остатки** (`sync-ozon-stocks`)
   - Актуальные остатки по FBO-складам Ozon
   - Синхронизирует в warehouse inventory

3. **Продажи** (`sync-ozon-sales`)
   - FBO-отправления
   - Статусы доставки
   - Информация о покупателе (анонимизированная)

4. **Возвраты** (`sync-ozon-returns`)
   - Возвратные отправления
   - Причины возвратов
   - Автоматическое списание со склада

5. **Финансовые операции** (`sync-ozon-transactions`)
   - Выплаты от Ozon
   - Комиссии, штрафы, бонусы
   - Логистические сборы
   - Автоматическое создание FinanceTransactions

### 3.2 AI-управление магазином

**MCP Tools (54+ методов Ozon API):**
- Оказывает полный доступ к Ozon Seller API через AI-агента
- Категории эндпоинтов (примеры):
  - Product Management (создание, обновление, удаление карточек)
  - Price Management (изменение цен через API)
  - Inventory Management (обновление остатков)
  - Order Management (работа с заказами)
  - Analytics (аналитика из Ozon)
  - Ratings & Reviews (отзывы)
  - и другие...

**Что может делать AI-агент:**
```
Примеры:
- "Поднять цену на все товары в категории 'гаджеты' на 10%"
- "Обновить остатки для всех товаров на складе"
- "Получить аналитику по продажам за месяц"
- "Создать новую карточку товара"
- "Установить минимальную цену для всех товаров"
- "Получить топ 10 товаров по выручке"
```

### 3.3 Обогащение интерфейса

**Middleware обогащения:**
1. `catalog-enrichment` — добавляет данные Ozon в карточку товара
2. `inventory-enrichment` — показывает остатки на каждом складе Ozon
3. `procurement-enrichment` — учитывает in-transit товары из Ozon
4. `sales-enrichment` — показывает комиссии Ozon, статусы доставки

**Column Documentation:**
- Пояснения полей (что такое "комиссия маркетплейса")
- Непосредственно в UI таблиц

### 3.4 Конфигурация
- Хранение API-ключей (Client ID, Client Secret)
- Привязка нескольких аккаунтов Ozon
- Расписание синхронизации (автоматическое)

## 4. Что означает "AI Agent First" в контексте этого продукта

### 4.1 Архитектурная парадигма

**AI Agent First** означает, что система спроектирована так, чтобы работать с AI-агентом как с основным интерфейсом, а не как с дополнением.

**Практическое выражение:**
1. **MCP-сервер встроен в ядро** (не плагин)
   - 54+ инструментов доступны любому AI-клиенту
   - REST API полностью отображается в MCP tools

2. **Инструменты максимально сложные**
   - Один вызов может выполнить много действий (создать заказ + распределить затраты)
   - Агент может анализировать и принимать решения на основе возвращаемых данных

3. **MCP Resources (документация в протоколе)**
   - Категории Ozon API доступны как ресурсы
   - Агент может читать документацию прямо в протоколе (не ходить на сайт)

### 4.2 Пример workflow'а через AI

**Сценарий: AI-агент проводит анализ и оптимизацию закупок**

```
1. Agent вызывает: list_products()
   ← получает список товаров

2. Agent вызывает: get_procurement_forecast()
   ← видит какие товары нужно заказать

3. Agent анализирует:
   - какие товары дешевле на 1688.com
   - какие склады переполнены
   - какие товары вообще не продаются

4. Agent вызывает: ozon_api (PUT /v3/product/prices)
   ← поднимает цены на товары с низкой скоростью продажи

5. Agent вызывает: create_procurement_order()
   ← создаёт заказ поставщику на нужные товары

6. Agent вызывает: get_finance_summary()
   ← проверяет экономику
```

### 4.3 Преимущества для пользователя

1. **Естественный язык вместо UI**
   - "Какие товары требуют переторговки?" (вместо 5 кликов по разным страницам)
   - "Оптимизируй остатки в соответствии с маржей" (вместо ручных правок)

2. **Автоматизация рутины**
   - Синхронизация, расчёт себестоимости — всё в фоне
   - AI может автоматически выполнять повторяющиеся задачи

3. **Лучшие решения**
   - AI может анализировать много параметров одновременно
   - Может предложить оптимизацию, которую человек не заметит

## 5. Какие workflows доступны пользователям

### 5.1 Core Workflows (встроенные)

#### 1. **Receive Order** (`receive-order.ts`)
Поступление товара от поставщика:
- **Входные данные**:
  - ID заказа поставщику
  - Количества по позициям (может быть меньше заказанного)
- **Что делает**:
  - Распределяет общие затраты (логистика, таможня) между товарами
  - Рассчитывает себестоимость каждой единицы с учётом всех расходов
  - Заносит товар в историю партий
  - Обновляет средневзвешенную себестоимость
- **Методы распределения**:
  - По цене товара (proportional)
  - По весу (for shipping)
  - Поровну (equal)
- **Особенность**: если товара пришло меньше, чем заказано:
  - Режим "redistribute" — абсорбирует стоимость незачислённого товара в полученный
  - Режим "ignore" — учитывает затраты только для полученного

#### 2. **Create Sale** (`create-sale.ts`)
Регистрация продажи:
- Поддержка разных источников (Ozon, manual, API)
- Автоматическое списание со склада (FIFO или по партиям)
- Расчёт себестоимости реализованного товара
- Создание финансовой транзакции

#### 3. **Return Sale** (`return-sale.ts`)
Обработка возврата:
- Возврат товара на склад
- Обратная запись в себестоимость
- Корректировка финансов

#### 4. **Write Off** (`write-off.ts`)
Списание товара (брак, кража, порча):
- Методы списания:
  - FIFO (старые партии первыми)
  - LIFO (новые партии первыми)
  - Average cost
- Финансовое отражение (убытки)

#### 5. **Unreceive Order** (`unreceive-order.ts`)
Откат поступления:
- Отмена принятого заказа
- Восстановление себестоимости
- Корректировка финансов

#### 6. **Initial Balance** (`initial-balance.ts`)
Первоначальное сальдо:
- Импорт старых остатков
- Запись в истории партий
- Начальная себестоимость

### 5.2 Workflows через API (для AI-агента)

**Через MCP tools, агент может:**
1. **Управление каталогом** — создание, редактирование, удаление товаров
2. **Управление закупками** — создание заказов поставщикам (в том числе с распределением затрат)
3. **Управление складом** — поступления, списания, перемещения
4. **Анализ** — получение отчётов по финансам, аналитике, юнит-экономике
5. **Интеграция с Ozon** — управление ценами, остатками, карточками товаров

### 5.3 Сложные сценарии

**Пример: "Отпустить товар на маркетплейс"**
1. Создать Master Card в каталоге
2. Связать с Ozon (через синхронизацию)
3. Получить из Ozon первоначальные остатки
4. Записать в историю партий себестоимость

**Пример: "Оптимизировать остатки по маржинальности"**
1. Получить все товары
2. Для каждого рассчитать маржу (из аналитики)
3. Товары с маржей < 15% — снизить остаток (меньше заказать)
4. Товары с маржей > 30% — увеличить остаток (больше заказать)

## 6. Архитектура (краткий обзор для впечатлений)

### 6.1 Tech Stack
```
Frontend:  React 19 + Vite 6 + Tailwind CSS 3 + TanStack Query 5
Backend:   Hono 4 (Node.js HTTP)
Database:  PostgreSQL 17 + MikroORM 6 (TypeScript ORM)
Auth:      Logto OIDC (cloud) или email+password (self-hosted)
AI:        MCP (Model Context Protocol) — встроенный в бэкенд
DI:        awilix (Dependency Injection)
Session:   iron-session (encrypted cookies)
```

### 6.2 Plugin Architecture
- **Convention-based discovery** — плагины находятся по структуре папок
- **Type-safe entities** — TypeScript + MikroORM
- **Auto-schema updates** — таблицы плагинов создаются автоматически
- **Extension points**:
  - Services (для бизнес-логики)
  - Routes (для API)
  - Middleware (для обогащения данных)
  - UI Pages (React компоненты)
  - MCP Tools & Resources (для AI)
  - Navigation items (в сайдбаре)

### 6.3 Database Schema
```
master_cards — основные товары
supplier_orders — заказы поставщикам
supplier_order_items — позиции заказов (с себестоимостью)
finance_transactions — операции (ДДС)
sales — продажи
ozon_integration — конфиг Ozon плагина
... + плагины добавляют свои таблицы
```

## Заключение

**MPFlow** — это не просто ERP, а платформа для управления маркетплейсами через AI. Она:
1. Решает реальную боль продавцов (себестоимость, прибыль)
2. Использует современные технологии (MCP, AI-агенты)
3. Open Source и расширяемая (плагины)
4. Красивая и удобная UI
5. Интегрирована с Ozon из коробки

Для видео демо лучше всего показать прогноз закупок (красивая визуализация) + управление через AI (впечатляет tech-аудиторию) + P&L отчёт (решает реальную проблему).
