# MPFlow — Итоговое резюме анализа

## 🎯 Суть продукта (в одном предложении)

**MPFlow** — это открытая AI Agent First система управления маркетплейсами (Ozon, Wildberries, Amazon FBA), которая считает прибыль как положено, используя техники управленческого учёта (себестоимость по партиям, распределение затрат) и позволяет управлять магазином через AI-агентов.

---

## 📊 Что показать в видео (ранжировано по впечатляющести)

### ⭐⭐⭐⭐⭐ Главные фичи (обязательно показать)

1. **Прогноз закупок с визуализацией** (90 сек)
   - Красивая timeline bar (красный/жёлтый/зелёный статус)
   - Автоматический расчёт нужного количества
   - Один клик → заказ поставщику
   - **Почему впечатляет**: визуально красиво, решает реальную боль

2. **Управление через AI** (60 сек)
   - Подключить Claude Code через MCP
   - Команда на русском: "Какие товары требуют переторговки?"
   - AI вызывает API, возвращает аналитику
   - **Почему впечатляет**: tech-аудиторию дует от идеи управления через речь

3. **P&L + Юнит-экономика** (60 сек)
   - Все комиссии рассчитаны автоматически
   - ROI по каждому товару
   - Видно какие товары убыточные
   - **Почему впечатляет**: решает реальную проблему (считать прибыль верно)

### ⭐⭐⭐⭐ Дополнительные фичи

4. **Синхронизация с Ozon в реальном времени** (45 сек)
   - Товары, остатки, продажи появляются автоматически
   - Финансы уже рассчитаны

5. **Распределение затрат при поступлении** (30 сек)
   - Таможня, логистика, упаковка распределяются по товарам
   - Методы: по цене, по весу, поровну

6. **Open Source + Self-hosted** (15 сек)
   - Docker Compose в 5 минут
   - Нет провайдерской зависимости

## 🚀 Как это использует пользователь

### Дневная рутина

```
9:00 — Проверить аналитику
"Какой товар принёс больше всего денег вчера?"
Claude Code: "Wireless Headphones — 15,000 ₽ выручки, 6,500 ₽ прибыли"

10:00 — Управление остатками
"Какие товары требуют срочного заказа?"
Claude Code показывает 3 товара в красной зоне (< 5 дней)
Пользователь нажимает "Create Order" → готово

12:00 — Проверить финансы
Откроет /analytics → P&L за месяц
Видит маржу 35% (реальную, с учётом всех затрат)

15:00 — Обновить цены
"Поднять цены на товары с маржей < 15% на 10%"
Claude Code проверяет какие товары это, вызывает Ozon API
Цены обновлены на 10 товарах

18:00 — Поступление от поставщика
Заказ в статусе "В пути" → меняет на "Received"
Система считает себестоимость каждого товара
Остатки обновились, финансы пересчитались
```

### Неделя

```
Понедельник:
- Проверить аналитику за неделю
- Если есть убыточные товары → снизить себестоимость или цену

Среда:
- Синхронизировать с Ozon
- Проверить возвраты, статусы доставки

Пятница:
- Запланировать закупки на следующие 2 недели
- Система показит что нужно заказать, когда, в каком количестве
- Нажать "Create Order" → готово
```
