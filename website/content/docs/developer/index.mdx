# Для разработчиков

MPFlow — открытая система на TypeScript. Один Node.js процесс: Hono API + React SPA + MikroORM + плагины.

## Архитектура

```
┌─────────────────────────────────┐
│   Браузер                       │
│   http://localhost:3000         │
└────────────┬────────────────────┘
             │
┌────────────▼────────────────────┐
│   MPFlow Admin (Node.js)       │
│   - Hono API (:3000)          │
│   - React SPA (встроена)      │
│   - Плагины (Ozon и др.)     │
├─────────────────────────────────┤
│   Core-модули:                  │
│   - master-card (каталог)      │
│   - supplier-order (закупки)   │
│   - finance (транзакции)       │
│   - sale (продажи)             │
├─────────────────────────────────┤
│   Плагины (convention-based):  │
│   - entities → auto-schema     │
│   - services → DI (awilix)    │
│   - routes → Hono API         │
│   - middleware → enrichment    │
└──────┬──────────────────────────┘
       │
┌──────▼──────┐
│ PostgreSQL  │
│ :5432       │
└─────────────┘
```

**Ключевые принципы:**
- **Один процесс** — API и SPA в одном Node.js, без микросервисов
- **Convention over configuration** — плагины обнаруживаются по структуре папок
- **Auto-schema** — таблицы плагинов создаются автоматически (`safe: true`, без DROP)
- **Request-scoped DI** — каждый HTTP-запрос получает свой EntityManager

## Быстрый старт для разработки

```bash
git clone https://github.com/teploe-odealko/mp-flow.git
cd mp-flow/admin
npm install

# Запустить PostgreSQL
docker run -d --name mpflow-pg \
  -e POSTGRES_USER=mpflow \
  -e POSTGRES_PASSWORD=mpflow \
  -e POSTGRES_DB=mpflow \
  -p 5432:5432 \
  postgres:17-alpine

# Dev-сервер
npm run dev
```

Админка: `http://localhost:5173` (клиент) с прокси на API `http://localhost:3000`.
